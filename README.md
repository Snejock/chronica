
# Chronica

**Chronica** — это проект для автоматизированного сбора, обработки и анализа новостей из различных RSS-источников. Система использует микросервисную архитектуру и современные инструменты обработки данных (AI/LLM) для создания структурированного хранилища мировых событий.

## Основные возможности

- **RSS Fetching**: Автоматический сбор новостей из настраиваемого списка источников (Интерфакс, РБК, Коммерсантъ, BBC, NYT и др.).
- **AI Processing**:
    - Перевод новостей с использованием LLM (через Ollama).
    - Генерация эмбеддингов для текстового контента.
- **Data Warehousing**:
    - Хранение сырых данных в PostgreSQL и ClickHouse.
    - Обработка потоков данных через Redpanda (Kafka-совместимая платформа).
    - Управление миграциями и схемами данных.
- **Infrastructure**: Полностью контейнеризированная среда (Docker Compose).

## Архитектура проекта

Проект разделен на несколько ключевых компонентов:

### Сервисы (`/services`)
- **rss-fetcher**: Сервис на Python, отвечающий за периодический опрос RSS-лент и сохранение первичных данных.

### Хранилище данных (`/dwh`)
- **PostgreSQL**: Основное хранилище для структурированных данных (схемы `stg`, `dtl`, `dm`).
- **ClickHouse**: Аналитическое хранилище для высокопроизводительных запросов.
- **Redpanda**: Шина сообщений для передачи данных между компонентами.
- **Minio**: S3-совместимое объектное хранилище.
- **Ollama**: Локальный запуск LLM моделей для обработки текста.
- **Redis**: Кэширование и управление очередями.

### Общие компоненты (`/common`)
- Общие модели данных (Pydantic), конфигурации и утилиты, используемые во всех сервисах.

## Технологический стек

- **Язык**: Python 3.14+
- **Менеджер пакетов**: [uv](https://github.com/astral-sh/uv)
- **Базы данных**: PostgreSQL, ClickHouse, Redis
- **Message Broker**: Redpanda
- **AI/LLM**: Ollama
- **Infrastructure**: Docker, Docker Compose, dbt (для трансформации данных)

## Начало работы

### Требования
- Docker и Docker Compose
- [uv](https://github.com/astral-sh/uv) (для локальной разработки)

### Запуск проекта
1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/your-username/chronica.git
   cd chronica
   ```

2. Запустите инфраструктуру и сервисы через Docker Compose:
   ```bash
   docker-compose up -d
   ```

3. Проверьте статус контейнеров:
   ```bash
   docker-compose ps
   ```

## Конфигурация
- Список RSS-каналов настраивается в файле `services/rss-fetcher/config/rss_feeds.yml`.
- Настройки подключения к базам данных и сервисам находятся в соответствующих `config.yml` и `.env` (если предусмотрены).

## Структура БД
Проект использует многоуровневую архитектуру данных (Data Vault элементы + Star Schema):
- **stg**: Сырые данные из источников.
- **dtl (Detailed)**: Очищенные и нормализованные данные (Хабы, Сателлиты).
- **dm (Data Mart)**: Витрины данных для конечных пользователей и аналитики.

Миграции находятся в папке `dwh/migrations`.
